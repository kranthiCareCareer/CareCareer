# CareCareer — Idempotency Contract

Version: 1.0
Status: Phase 0 Architecture Lock
Date: 2026-07-16

---

## 1. Purpose

Retries will occur from mobile clients (offline sync), queue consumers,
portal interactions, and migration connectors. Every externally-initiated
mutation MUST be idempotent. This contract defines how.

---

## 3. Rules

### 3.1 HTTP API Idempotency

1. **`Idempotency-Key` is REQUIRED** for all externally-initiated POST/PATCH mutations.
2. **Key scope:** `{tenantId}:{endpoint}:{idempotencyKey}` — same key on different
   endpoints or tenants are independent.
3. **Same key + same payload** → returns the original response (status code + body).
4. **Same key + different payload** → returns `409 IDEMPOTENCY_CONFLICT`.
5. **Key storage duration:** 7 days minimum (covers retry windows, weekend delays).
6. **Key format:** Client-chosen string, 1-128 characters. UUIDs recommended.
7. **Response includes:** The original response is stored and replayed exactly.

### 3.2 Queue/Event Consumer Idempotency

1. **Consumers deduplicate by `eventId`** (from event envelope).
2. **Inbox pattern:** Consumer records processed eventIds in a local table.
3. **Duplicate delivery:** Returns success without re-processing.
4. **Ordering:** Per-aggregate ordering is guaranteed by the queue infrastructure.
   Consumers MUST handle at-least-once delivery within that ordering.

### 3.3 Domain Aggregate Idempotency

1. **Version checks:** Commands include expected `version` for optimistic concurrency.
2. **Stale commands rejected** with `VERSION_CONFLICT` error.
3. **State machine guards:** A command for an already-completed transition is idempotent
   (returns current state without error).

### 3.4 Clock Event Idempotency

Clock events have special handling due to offline mobile submission:

1. **`deviceEventId`** is generated by the mobile device at clock time.
2. **Deduplication scope:** `{tenantId}:{assignmentId}:{deviceEventId}`
3. **Offline behavior:** Device stores event locally; syncs when connected.
4. **Multiple submissions of same deviceEventId:** Returns the original response.
5. **Server timestamp** recorded separately from device timestamp to detect drift.
6. **Out-of-order delivery:** Accepted if within allowed window; otherwise rejected.

### 3.5 Legacy Import Idempotency

Migration imports have natural keys:

1. **Deduplication scope:** `{tenantId}:{externalSystem}:{entityType}:{externalId}`
2. **Re-import of same record:** Updates if changed; no-op if unchanged.
3. **Conflict on different CareCareer entity:** Returns `EXTERNAL_REFERENCE_CONFLICT`.
4. **Import operations are always safe to retry.**

### 3.6 Financial Calculation Idempotency

1. **Deduplication scope:** `{tenantId}:{timecardId}:{timecardVersion}:{ruleSetVersion}`
2. **Same timecard version + same rules** → returns existing calculation.
3. **Different timecard version** → creates new calculation; supersedes previous.
4. **Different rule version** → creates new calculation; supersedes previous.
5. **Completed calculations are NEVER mutated** — only superseded.

## 4. Implementation

### 4.1 Idempotency Store Schema

```sql
CREATE TABLE idempotency_keys (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,
    key_scope VARCHAR(500) NOT NULL,
    idempotency_key VARCHAR(128) NOT NULL,
    request_hash VARCHAR(64) NOT NULL,
    response_status INTEGER NOT NULL,
    response_body JSONB NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    expires_at TIMESTAMPTZ NOT NULL,

    CONSTRAINT uq_idempotency_key
        UNIQUE (tenant_id, key_scope, idempotency_key)
);

-- RLS
ALTER TABLE idempotency_keys ENABLE ROW LEVEL SECURITY;
ALTER TABLE idempotency_keys FORCE ROW LEVEL SECURITY;
CREATE POLICY tenant_isolation ON idempotency_keys
    USING (tenant_id = current_setting('app.tenant_id')::UUID);

-- TTL cleanup
CREATE INDEX idx_idempotency_expires ON idempotency_keys (expires_at);
```

### 4.2 Middleware Pseudocode

```typescript
async function idempotencyMiddleware(req, res, next) {
  const key = req.headers['idempotency-key'];
  if (!key) return res.status(400).json({ error: { code: 'IDEMPOTENCY_KEY_REQUIRED' } });

  const scope = `${req.tenantId}:${req.method}:${req.path}`;
  const requestHash = hash(req.body);

  const existing = await findIdempotencyRecord(scope, key);

  if (existing) {
    if (existing.requestHash !== requestHash) {
      return res.status(409).json({ error: { code: 'IDEMPOTENCY_CONFLICT' } });
    }
    return res.status(existing.responseStatus).json(existing.responseBody);
  }

  // Process the request
  const result = await next();

  // Store the result
  await storeIdempotencyRecord(scope, key, requestHash, result.status, result.body);

  return result;
}
```

## 5. Client Guidance

### For Portal/Web Clients

- Generate a UUID for each user action (button click, form submit)
- Store the key locally until response received
- On network timeout: retry with the SAME key and SAME payload
- On success: discard the key

### For Mobile Clients (Offline)

- Generate `deviceEventId` at the moment of the action (clock tap)
- Store event in local queue with the deviceEventId
- When online: submit all queued events with their original deviceEventIds
- Server deduplicates by deviceEventId — safe to retry indefinitely

### For Migration Scripts

- Use `{system}:{entityType}:{externalId}` as the natural idempotency key
- Safe to re-run entire import batches (idempotent by design)
- Conflicts surface as errors, not silent overwrites

## 6. Expiration and Cleanup

| Record Type                | TTL                                 | Cleanup                              |
| -------------------------- | ----------------------------------- | ------------------------------------ |
| HTTP idempotency keys      | 7 days                              | Scheduled job deletes expired rows   |
| Event inbox records        | 30 days                             | Scheduled job archives then deletes  |
| Clock event deviceEventIds | 48 hours (within clock-event table) | Part of clock event record lifecycle |
| Legacy import records      | Permanent (in external_references)  | Never deleted                        |
